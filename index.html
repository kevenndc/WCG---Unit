O que é o World Community Grid?

World Community Grid (WCG)s é um projeto da IBM que pretende criar o maior supercomputador  público do mundo para realizar pesquisas científicas que beneficiem a humanidade.

O conceito do World Community Grid é que qualquer um pode participar ativamente de uma pesquisa científica que busque drogas que combatam câncer, AIDS ou a dengue sem ter quaisquer conhecimentos profundos sobre biologia e medicina. O requisito mínimo para participar de qualquer pesquisa é se cadastrar no site do projeto e fazer o download da plataforma BOINC, que é usada para fazer a intercessão entre o computador do voluntário e o servidor central.

Como funciona?

A ideia por trás do projeto é simples, o computador do voluntário é usado para processar dados que, normalmente, demorariam muito tempo para serem processados por um único computador. Quando se soma os computadores e os dados processados por eles ao redor de todo o mundo, pesquisas que demandam muita computação e dados que normalmente poderiam levar anos para serem processados, podem ser processados em semanas ou meses.

A plataforma BOINC utiliza apenas processamento ocioso do computador do voluntário, que é usado para processar dados e devolvê-los ao servidor central. Desse modo, aplicação não interfere na utilização diária do computador.

Este modelo é chamado de computação filantrópica ou voluntária. O ambiente de computação voluntária apresenta características especiais. Uma delas é que a disponibilidade das máquinas é extremamente dinâmica, com usuários entrando e saindo sem aviso prévio, e por isso o WCG, através do BOINC, implementa uma API para função de checkpoint, que permite que o estado de execução da aplicação seja salvo e retomado no mesmo ponto em que foi interrompido.

A arquitetura do BOINC também é simples, há um banco de dados relacional ao lado do servidor, que armazena as informações referentes a cada projeto, como

usuários cadastrados, unidades de trabalho disponíveis, unidades de trabalho enviadas e processadas e assim por diante. Cada projeto de pesquisa também tem um serviço responsável por distribuir as unidades de trabalho e tratar os resultados recebidos. Os servidores de dados são os responsáveis pela distribuição dos arquivos de dados e seu recebimento. Os serviços de escalonamento controlam o fluxo de entrega das unidades de trabalho aos usuários, conforme a produtividade de cada um. A interface com o usuário é feita via Web.

O WCG, como é baseado no BOINC, e como qualquer projeto de computação filantrópica, opera com o servidor enviando dados aos programas cliente apenas por solicitação destes, e estes, após executar as computações, retornam os resultados ao servidor.

Há algum risco?

As questões de segurança merecem atenção para impedir falsificação de resultados, os projetos do WCG utilizam redundância para diminuir a chance de ocorrências de erros e falsificações. Isto é, cada unidade de trabalho é distribuída para múltiplos clientes, os resultados gerados por eles são comparados e são aceitos apenas aqueles nos quais o consenso é obtido. Caso necessário, novos clientes são adicionados para fazer o processamento dos mesmos dados, a fim de se analisar os resultados quando não há um consenso.

O WCG gerencia a maior parte das tarefas de redundância, embora a aplicação deva se envolver com alguns dos processos de validação e assimilação. Validação acontece quando um número suficiente de respostas chega ao servidor e devem ser avaliadas a fim de se provar um consenso. Assimilação é o mecanismo no qual o projeto é notificado do término, com sucesso ou não, de uma unidade de trabalho. Quando é obtido consenso, os resultados são enviados para processamento complementar pela parte da aplicação que roda no servidor. Se a unidade de trabalho não conseguir ser executada com sucesso, por não haver quórum ou não ter sido obtido consenso, a aplicação no servidor é acionada para determinar ações alternativas.

O WCG utiliza assinatura de código para eliminar a possibilidade de um cracker distribuir um vírus no lugar do cliente verdadeiro. Cada projeto possui um par de chaves criptográficas que são utilizadas para autenticar os programas distribuídos.
